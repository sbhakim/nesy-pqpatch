# Evaluation corpora

Three tiers with different ground-truth guarantees, plus an adversarial trap
suite. The distinction matters: claims about detection recall are only made
where ground truth is exact.

**`tier1/`** — benchmark-derived cases with reference migrations. Every case
exists in two surfaces: `original/` and a semantics-preserving `mutated/`
variant regenerated by `mutate.py` (identifiers renamed, files restructured,
literals neutralized). Results on the two surfaces are reported side by side;
a gap between them measures benchmark memorization, and is a finding rather
than noise. Never edit `mutated/` by hand.

**`tier2/`** — purpose-built reference applications with exact ground truth.
Each application directory contains a `sites.yaml` enumerating every seeded
vulnerable site, including sites deliberately constructed to defeat
pattern-based detection (configuration-driven algorithm names, reflection).
Those are labeled `detectable: false`; the detector is *supposed* to miss
them, and the miss rate is a measured quantity, not a defect.

**`tier3/`** — open-source projects, pinned by commit in `manifest.yaml`
rather than vendored. Permissive licenses only. Ground truth here is sampled
and annotator-labeled, not exhaustive.

**`traps/`** — scenarios engineered so that the plausible completion is
unsafe. `dev/` traps inform rule development; `heldout/` traps are authored
only after the rule set is frozen and are locked by a CI guard from that
point on. The pre-registered primary endpoint of the evaluation is computed
on the held-out subset exclusively — reported co-primary with the full-suite
RUA for statistical power (`metrics.dual_rua`).

The trap format is `traps/SCHEMA.md` (v2): each trap carries its provenance
(taxonomy vs. external PR/CVE), two blind annotator labels (Cohen's κ), and a
difficulty control (does the unsafe patch compile; would build+test alone catch
it) so the load-bearing catch can be isolated from what a compile gate handles
anyway. The held-out suite is intentionally unfrozen while the rule set is still
growing (6 of 14 L1 rules; L2 pending ADR-001): freezing traps against an
unfinished taxonomy would overfit them to it.
